# 小狐狸 AI 伙伴 — 技术架构执行摘要

> 价值主张：让孩子在3分钟内产生"它是我的"的感觉，建立情感连接

---

## 一、核心目标

| 目标 | 描述 |
|------|------|
| 情感连接 | 3分钟内让孩子产生"它是我的"的感觉 |
| 语音延迟 | P95 < 2.5秒（硬指标） |
| 目标用户 | 4-7岁儿童（4岁需验证） |
| MVP范围 | 第一次见面流程 + 每日见面流程 |

---

## 二、关键技术决策

| 组件 | 选型 | 理由 |
|------|------|------|
| 前端 | Vue3 SPA | 团队熟悉，快速验证 |
| 后端 | Express.js | 同语言全栈，WebSocket原生支持 |
| 数据库 | SQLite | 零配置，MVP规模足够 |
| 缓存 | Redis | 实时状态管理，TTL=30min |
| ASR | Whisper本地 | 儿童普通话识别好，延迟<300ms |
| TTS | Edge TTS | 免费，中文童声好，延迟<500ms |
| LLM | GPT-4o-mini | 儿童对话质量好，延迟<800ms |
| 部署 | Docker单机 | MVP规模极小，快速部署 |

---

## 三、核心架构

### 语音交互管道（延迟预算）

```
Child voice → WebAudio → WebSocket → ASR(300ms) → LLM(800ms) → TTS(500ms) → WebSocket → Playback
                                                                                 ↓
                                                                     端到端P95: ~1800ms ✅
```

### 对话引擎架构

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  FSM Router  │ ──→ │  LLM Generator│ ──→ │  Emotion     │
│  5-step FSM  │     │  Prompt模板  │     │  Adjuster    │
│  daily_loop  │     │  + context   │     │  密度/脆弱度 │
└──────────────┘     └──────────────┘     └──────────────┘
         │                                         │
         └─────────────────┬───────────────────────┘
                           ▼
                   ┌──────────────┐
                   │ Interest     │
                   │ Brancher     │
                   │ 名字→兴趣分型 │
                   └──────────────┘
```

## 四、关键API

| 协议 | 路径 | 说明 |
|------|------|------|
| WS | `/ws/voice/{child_id}` | 语音双向流（核心） |
| POST | `/api/profile` | 创建/更新画像 |
| POST | `/api/session/start` | 开始会话 |
| POST | `/api/session/end` | 结束会话，输出数据 |
| GET | `/api/story/{child_id}` | 获取故事进度 |
| POST | `/api/borrow/check` | 借分契约状态检查 |

## 七、三状态分类器（不想学时）

| 状态 | 关键词 | 行为 |
|------|--------|------|
| tired | 不想玩、困了、累了 | 不递增计数器，结束会话 |
| difficult | 太难、不会、不懂 | 不递增计数器，降低难度 |
| rebellious | 我不要学、不想学、无聊 | 递增计数器，触发借分契约 |
